Présentation du code - Voyageur de Commerce

But du projet

Le but est de résoudre des instances du problème du voyageur de commerce (TSP) à partir de fichiers au format .tsp contenant les coordonnées ou distances entre les villes. L'objectif est de trouver un chemin passant une seule fois par chaque ville et revenant à la ville de départ, tout en minimisant la distance totale.

Structure du programme

Le programme est composé de plusieurs parties :

  • Lecture du fichier TSP : extraction des coordonnées ou distances entre villes.
  • Génération d'une solution aléatoire : création d'un chemin aléatoire.
  • Optimisation par 2-Opt : amélioration du chemin par échange d'arêtes.
  • Comparaison et sauvegarde : on garde la meilleure solution trouvée parmi plusieurs essais.

Structures principales

1. DonneesTSP

  • Contient toutes les données de l'instance TSP (dimensions, distances, coordonnées, nom de l’instance, etc.).

2. Solution

  • Représente une solution avec un chemin (ordre des villes) et la distance totale.

Lecture et allocation des données

La fonction lireFichierTSP permet de lire un fichier .tsp et de remplir les structures en fonction du type d'information (soit des distances, soit des coordonnées à convertir en distances). Les fonctions initialiserDonnees, allouerMemoire et libererMemoire assurent une bonne gestion de la mémoire.

Génération d'une solution aléatoire

On utilise l'algorithme Fisher–Yates pour mélanger aléatoirement l'ordre des villes, via la fonction genererSolutionAleatoire.

Calcul de la distance d'une solution

La fonction calculerDistance parcourt le chemin et additionne les distances entre chaque ville consécutive.

Optimisation locale avec 2-Opt

Le 2-Opt est une méthode d'amélioration qui consiste à inverser des sous-chemins si cela réduit la distance totale. Elle est implémentée dans la fonction optimisation2Opt.

Exécution principale (main)

Dans la fonction main, le programme :

  • Lit les données depuis un fichier TSP (par exemple bayg29.tsp ou att48.tsp).
  • Réalise plusieurs essais (10 000 par défaut).
  • Pour chaque essai :
    • Génère une solution aléatoire.
    • La corrige avec l'algorithme 2-Opt.
    • Compare avec la meilleure solution trouvée jusqu'ici.
  • Affiche et sauvegarde la meilleure solution dans un fichier .tour.

Conclusion

Ce projet met en œuvre des techniques classiques de résolution du TSP : génération aléatoire, amélioration par heuristique (2-Opt), et gestion mémoire dynamique. Le programme est générique et peut lire tout fichier TSP compatible TSPLIB.
